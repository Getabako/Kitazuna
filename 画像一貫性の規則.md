## 1. プロジェクト概要

### 1.1 プロジェクトビジョン
- **テーマ:** 伝統と現代、AI×レトロの融合
- **舞台:** 秋田県全域（男鹿、湯沢、秋田市、三湖、大館、鹿角）
- **ターゲット:** レトロゲームファンと秋田観光に興味がある層
- **特色:** 秋田県の名所・文化・伝統を実在のロケーションとともに紹介する聖地巡礼型ゲーム

### 1.2 キャラクターコンセプト
秋田県の名物・文化・伝承をモチーフにした個性豊かなキャラクターたち：

- **杉野健太（スギノ・ケンタ）:** 内向的な大学生主人公
- **なぎさ（長砂）:** なまはげをモチーフにした守護者

- **稲庭（イナニワ）:** 稲庭うどんをモチーフにしたキャラ
- **たんぽ（タンポ）:** きりたんぽをモチーフにしたキャラ
- **八郎（ハチロウ）:** 八郎太郎伝説をモチーフにした龍神族
- **辰子（タツコ）:** 辰子姫伝説をモチーフにした龍姫
- **関東（カントウ）:** 竿灯祭りをモチーフにしたキャラ

- **スギノオウ:** ラスボス的存在、未来杉の化身

## 2. アートスタイル全体方針

### 2.1 基本方針
**「レトロゲームの温かみ × 現代的なクオリティ」**

1. **ノスタルジックなレトロ感**
   - ファミコン・スーパーファミコン時代の親しみやすさ
   - くにおくんシリーズのデザイン言語を継承
   - ドット絵の制約を活かした表現

2. **現代的なブラッシュアップ**
   - 高解像度での表現力向上
   - スムーズなアニメーション
   - 豊かな色彩表現

3. **秋田らしさの表現**
   - 実在する建築物・風景の忠実な再現
   - 伝統工芸・祭りの意匠の取り入れ
   - 地域性を感じさせるディテール

### 2.2 統一すべき要素
- **視点:** 基本的に正面向き、または斜め45度の見下ろし視点
- **デフォルメ度:** くにおくんスタイルの2.5～3頭身
- **ライティング:** 明るく親しみやすい光源設定
- **質感:** セルアニメ調のシンプルで明快な塗り

## 3. キャラクターデザイン規則

### 3.1 制作段階別ガイドライン

#### 【Stage 1: ラフスケッチ】
**目的:** アイデア出し、コンセプト確認

- 手描きスケッチでOK（ノート・スケッチブック推奨）
- 複数ポーズ・表情のバリエーション探索
- モチーフの特徴を誇張して表現
- キャラクター名とモチーフを明記（例: 「タンポ」「きりたんぽ」）
- 日付とバージョン番号を記載

**参考例:**
- `hachiro0-1.jpg` - 八郎太郎の初期ラフ、力強い和風戦士スタイル
- `tanpo0-1.jpg` - きりたんぽモチーフ、コミカルな表情

#### 【Stage 2: クリーンアップ（線画）】
**目的:** デザインの確定、ディテールの詰め

- 線の太さは均一に（主線: 太め、内部線: 細め）
- 輪郭線は明確にクローズ（塗りつぶし対応のため）
- モチーフの象徴的要素を明確に配置
  - なぎさ: なまはげの面・包丁・藁装束
  - 秋田: 秋田犬の耳・尻尾・渦巻き模様
  - 小町: 扇・着物・編笠
- 構造的な整合性の確認（関節・身体バランス）

#### 【Stage 3: カラーデザイン】
**目的:** キャラクターの印象決定、配色設計

**配色原則:**
1. **メインカラー（40-50%）**
   - キャラクターの印象を決定づける色
   - モチーフの連想色を採用
     - なぎさ: 青（なまはげの鬼面）
     - 小町: 赤/ピンク（優雅さ・美しさ）
     - 秋田: オレンジ/ベージュ（秋田犬の毛色）
     - たんぽ: 茶色/ベージュ（米・きりたんぽ）

2. **サブカラー（30-40%）**
   - メインカラーを引き立てる補色または類似色
   - 秋田の伝統色を積極的に採用
     - 藍色（藍染）、茜色、山吹色

3. **アクセントカラー（10-20%）**
   - 視線を誘導する差し色
   - 紅白の縄、金色の装飾など

**色彩ガイド:**
- **彩度:** 中～高彩度（レトロゲームらしい鮮やかさ）
- **明度:** 明るめ基調（親しみやすさ重視）
- **コントラスト:** 明確な明暗差で視認性確保
- **グラデーション:** 最小限（ドット絵化を考慮）

#### 【Stage 4: 完成カラーイラスト】
**仕様:**
- **解像度:** 最低1000px以上の縦サイズ推奨
- **ファイル形式:** PNG（透過背景推奨）
- **塗り方:** セルアニメ調、明確なエッジ
- **シェーディング:** 1～2段階のシンプルな影
- **ハイライト:** 控えめに配置
- **背景:** 透過PNG、または白背景

**命名規則:**
```
{キャラクター名}stand/{キャラクター名}stand.png
例: nagisastand/nagisastand.png
```

### 3.2 キャラクター別デザイン要素

#### なぎさ（長砂）- なまはげモチーフ
- **頭部:** 青いなまはげ面（赤髪、角2本）
- **武器:** 出刃包丁（右手）、提灯付き木棒（左手）
- **装束:** 青い着物、藁製の肩当て・腰巻き・脚巻き
- **配色:** 青（メイン）、黄（藁）、赤（差し色）
- **性格表現:** 豪快、頼もしい、伝統の守護者

#### 小町（コマチ）- 小野小町モチーフ
- **頭部:** 黒髪ツインテール、編笠
- **装備:** 扇子（白）、赤い着物
- **配色:** 赤/ピンク（メイン）、白（サブ）、金（アクセント）
- **性格表現:** 活発、元気、美しさと強さの両立

#### 秋田（アキタ）- 秋田犬モチーフ
- **頭部:** 秋田犬の耳、鼻
- **身体:** 擬人化された犬、渦巻き模様（尻尾・体側）
- **装束:** 赤いバンダナ、青い袴
- **配色:** オレンジ/ベージュ（メイン）、白（サブ）、赤（アクセント）
- **性格表現:** 忠実、優しい、頼れる仲間

#### たんぽ（タンポ）- きりたんぽモチーフ
- **頭部:** きりたんぽ型の帽子または髪飾り
- **装備:** きりたんぽを持つ、お玉やフライ返し
- **配色:** 茶色/ベージュ（メイン）、白（エプロン）、オレンジ（差し色）
- **性格表現:** 陽気、料理上手、温かみ

#### 八郎（ハチロウ）- 八郎太郎伝説モチーフ
- **頭部:** 龍の角、鋭い目つき
- **装束:** 和風鎧、装飾的な肩当て
- **配色:** 青緑（龍）、金（鎧）、白（サブ）
- **性格表現:** 威厳、力強さ、伝説の守護神

### 3.3 表情差分
各キャラクター最低5種類の表情を用意：

1. **通常（淡）:** デフォルト、落ち着いた表情
2. **喜び（喜）:** 笑顔、目を細める
3. **悲しみ（憂）:** 困った顔、涙目
4. **怒り（怒）:** 眉を吊り上げる、鋭い目つき
5. **驚き（困）:** 目を見開く、口を開ける

**その他感情:**
- **決意（決）:** 真剣な表情
- **愛情（愛）:** 優しい笑顔、照れ
- **希望（希）:** 明るく前向き
- **知識（知）:** 考え込む、説明する

## 4. ドット絵規則

### 4.1 基本仕様
- **キャンバスサイズ:** 32×32px ～ 64×64px（キャラクターサイズに応じて調整）
- **色数制限:** 1キャラクターあたり8～16色程度
- **アウトライン:** 黒または濃色で明確に
- **アンチエイリアス:** 最小限、ドット感を残す

### 4.2 アニメーションフレーム

#### 【必須アニメーション】
1. **Idle（待機）:** 2～4フレーム
   - 呼吸や軽い揺れの表現
   - ループアニメーション
   - 例: `nagisaidle1.png`, `nagisaidle2.png`, `nagisaidle3.png`

2. **Walk（歩行）:** 2～4フレーム
   - 足の動き、腕の振り
   - 体の上下動
   - 例: `nagisawalk1.png`, `nagisawalk2.png`

3. **Attack（攻撃）:** 3～5フレーム
   - 予備動作 → 攻撃 → 戻り
   - 武器の軌跡表現
   - 例: `nagisaattack1.png`, `nagisaattack2.png`, `nagisaattack3.png`

#### 【オプションアニメーション】
- **Jump（ジャンプ）:** 1～3フレーム
- **Damage（被ダメージ）:** 2～3フレーム
- **Special（必殺技）:** 4～8フレーム

### 4.3 ドット絵変換プロセス
1. 完成カラーイラストを参照
2. 特徴的なシルエットを優先
3. 色数を段階的に削減
4. ディテールを省略しつつ、モチーフの象徴要素は残す
   - なぎさ: 面の角、包丁、藁の質感
   - 小町: ツインテール、扇子
   - 秋田: 耳、渦巻き模様
5. 動きのあるポーズでテスト

### 4.4 命名規則
```
{キャラクター名}{アニメーション種類}/{キャラクター名}{アニメーション種類}{番号}.png

例:
nagisaidle/nagisaidle1.png
nagisawalk/nagisawalk1.png
nagisaattack/nagisaattack1.png
```

## 7. 制作フロー

### 7.1 新キャラクター制作プロセス

```
1. 企画・コンセプト設計
   ↓
2. モチーフリサーチ（秋田の文化・名物調査）
   ↓
3. ラフスケッチ（複数案）
   ├─ {キャラ名}0-1.jpg
   ├─ {キャラ名}0-2.jpg
   └─ {キャラ名}0-3.jpg
   ↓
4. デザイン確定・クリーンアップ
   ↓
5. カラーデザイン
   └─ {キャラ名}stand/{キャラ名}stand.png
   ↓
6. ドット絵変換
   ├─ {キャラ名}idle/{キャラ名}idle1.png
   ├─ {キャラ名}walk/{キャラ名}walk1.png
   └─ {キャラ名}attack/{キャラ名}attack1.png
   ↓
7. アニメーション実装
   ↓
8. ゲーム内テスト
   ↓
9. 最終調整
```

### 7.2 背景・マップ素材制作プロセス

```
1. ロケーション選定（秋田県内の実在地）
   ↓
2. 資料収集（写真、地図、文献）
   ↓
3. ラフレイアウト
   ↓
4. 線画作成
   ↓
5. カラーリング
   ↓
6. ドット絵化（必要に応じて）
   ↓
7. ゲーム内配置テスト
   ↓
8. 最終調整
```

### 7.3 バージョン管理
- **ラフ段階:** `{キャラ名}0-1.jpg`, `{キャラ名}0-2.jpg`...
- **確定版:** `{キャラ名}stand.png`
- **ドット絵:** `{キャラ名}{アニメーション}{番号}.png`

## 8. 品質チェックリスト

### 8.1 キャラクターデザイン
- [ ] モチーフの特徴が一目でわかる
- [ ] くにおくんスタイルの2.5～3頭身デフォルメ
- [ ] メインカラー、サブカラー、アクセントカラーのバランスが取れている
- [ ] シルエットだけで誰かわかる特徴的な形状
- [ ] 秋田らしい要素（伝統色、文様、装飾）が含まれている
- [ ] 表情差分が最低5種類用意されている
- [ ] 他キャラクターとの色被りがない（または差別化されている）
- [ ] 透過PNGで背景が正しく処理されている

### 8.2 ドット絵
- [ ] 元のカラーデザインの特徴を保持している
- [ ] アウトラインが明確
- [ ] 色数が適切（8～16色程度）
- [ ] アニメーションがスムーズにループする
- [ ] ファイル命名規則に従っている
- [ ] 32×32px または 64×64px のキャンバスサイズ
- [ ] 透過PNG形式

### 8.3 背景・マップ素材
- [ ] 実在するロケーションの特徴を捉えている
- [ ] 資料（写真・文献）に基づいた正確性
- [ ] プロジェクト全体のアートスタイルと統一されている
- [ ] 視点（俯瞰・水平）が統一されている
- [ ] 色調が明るく親しみやすい
- [ ] ディテール密度が適切（過密でも過疎でもない）
- [ ] ファイル命名規則に従っている

### 8.4 色彩
- [ ] プロジェクトカラーパレットから選択されている
- [ ] コントラストが十分（視認性確保）
- [ ] 感情色システムに準拠（会話シーン）
- [ ] レトロゲームらしい鮮やかさ
- [ ] 秋田の四季や文化を反映

### 8.5 技術仕様
- [ ] ファイル形式が正しい（PNG推奨）

## 10. よくある質問（FAQ）

### Q1. 新しいキャラクターを追加する場合、どこから始めればいいですか？
**A:** まず秋田県の文化・名物・伝説から適切なモチーフを選定します。その後、ラフスケッチで複数案を描き、チームでレビューしてから詳細デザインに進んでください。

### Q2. ドット絵のサイズはどれくらいが適切ですか？
**A:** キャラクターの場合、32×32px または 64×64px が推奨です。大型ボスや特殊キャラクターの場合は 128×128px まで拡大可能です。

### Q3. 色数制限は厳守すべきですか？
**A:** ドット絵の場合は8～16色を推奨しますが、絶対ではありません。レトロ感とクオリティのバランスを重視してください。

### Q4. 既存キャラクターのデザイン変更は可能ですか？
**A:** マイナーな調整（色の微調整、小物の追加）は可能ですが、大幅な変更は既存素材との整合性を損なう可能性があるため、チーム全体で協議してください。

### Q5. 実在しない架空のロケーションを追加できますか？
**A:** 基本的には実在する秋田県のロケーションを優先しますが、ストーリー上必要な場合は、秋田らしさを損なわない範囲で架空のロケーションを追加可能です。
